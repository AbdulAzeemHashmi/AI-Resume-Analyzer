# Contributing Guide

## 🌿 Branch Naming
- `feat/feature-name` - New features
- `fix/bug-description` - Bug fixes
- `docs/change-description` - Documentation changes
- `chore/task-description` - Maintenance tasks

## 📝 Commit Messages
Use clear, descriptive messages:
- ✅ `feat: add file upload endpoint`
- ✅ `fix: resolve PDF parsing error`
- ❌ `update`, `changes`, `final fix`

## 🔀 Pull Request Process
1. Create an issue first
2. Create a branch from the latest `main`
3. Keep PRs small and focused
4. Link the issue: `Closes #number`
5. Request a review
6. Address all feedback
7. Rebase before merging

## 🧪 Testing
- Run tests: `pytest`
- Run linter: `flake8`
- Manual testing checklist in PR
